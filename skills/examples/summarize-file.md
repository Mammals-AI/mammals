description: Summarize a file or codebase
---

When asked to summarize a file or project:

1. Read the file(s) requested
2. Identify the purpose, key functions/sections, and dependencies
3. Give a brief summary (3-5 sentences) at the top
4. If it's code, mention:
   - What language/framework
   - Main entry point
   - Key functions or classes
   - External dependencies
5. If it's a document, extract the main points

Keep it concise. Offer to dive deeper into specific parts.
